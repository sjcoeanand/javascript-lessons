  • Hoisting

    Basicallly, when Javascript compiles all of your code, all variable declarations using var are hoisted/lifted to the top of their functional/local scope (if declared inside a function) or to the top of their global scope (if declared outside of a function) regardless of where the actual declaration has been made. This is what we mean by “hoisting”.

    Example :

    ( What we actually write )

    console.log(myProfile) ;
    var myProfile = "front end developer" ;

    ( How compiler compiles above code )

    var myProfile ;
    console.log(myProfile) ;
    myProfile = "front end developer" ;

    Here // for var console throws error "undefined"
    and
    Here // for let or const console throws error "Reference Error : myProfile is not defined"

    This above scenario also valid for functions
